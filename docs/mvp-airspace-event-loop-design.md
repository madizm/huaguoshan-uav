# MVP 低空空域事件处置闭环设计

## 1. 目标

MVP 验证一个统一的 **低空空域事件处置闭环**：平台从多源输入中形成可解释的空域事件，完成发现、研判、预警、处置建议、认领处理、关闭和复盘。

MVP 包含两个场景，但共用同一套事件队列和工作台：

1. **禁飞/管控区入侵事件闭环**
2. **无人机林火巡查预警闭环**

两类事件通过事件类型区分，不拆成两个独立模块。

## 2. 已确认架构决策

相关 ADR：

- `docs/adr/0001-use-airspace-grid-as-core-spatial-index.md`
- `docs/adr/0002-use-situation-snapshots-and-event-history-first.md`
- `docs/adr/0003-use-shared-airspace-event-workbench-for-mvp-scenarios.md`

核心决策：

- 北斗/GGER **空域网格** 是核心空间索引与态势聚合单元，不只是 Cesium 展示图层。
- 第一阶段采用 **态势快照 + 事件历史**，不先建设全量实时流平台。
- 禁飞/管控区入侵与林火疑似预警共用 **空域事件处置工作台**。
- 外部输入先进入 **原始观测**，再形成 **融合判断** 和业务对象。
- MVP 不自动触发真实第三方反制动作，只支持观测、建议和模拟联动。

## 3. MVP 范围

### 3.1 包含

#### 场景 A：禁飞/管控区入侵事件闭环

给定一个已配置的禁飞/管控区和一条空域目标轨迹，系统能够：

1. 显示空域约束范围和空域网格。
2. 播放或导入一条目标航迹。
3. 通过空域网格粗筛可能命中的约束。
4. 使用 PostGIS 做精确入侵判定。
5. 通过入侵确认窗口生成正式空域事件。
6. 计算风险等级和处置优先级。
7. 展示证据、影响范围、命中网格和高度基准。
8. 匹配处置能力覆盖，或说明无可用能力。
9. 生成处置方案候选集。
10. 支持事件认领、人工处置结果、关闭依据和事件历史。

#### 场景 B：无人机林火巡查预警闭环

给定一个巡查任务、一架无人机资产和巡查观测，系统能够：

1. 显示巡查责任区、计划/实际航迹和空域网格。
2. 接收或模拟热红外热点、可见光烟点、人工标注等火点线索。
3. 将火点线索作为原始观测进入融合判断。
4. 通过火点确认窗口生成林火疑似预警事件。
5. 使用林火风险分级规则计算风险等级和处置优先级。
6. 展示证据、影响范围、环境研判因子和附近应急能力。
7. 生成复核与通知类处置方案候选集。
8. 支持人工确认、排除、升级处置或关闭，并记录决策证据链。

### 3.2 不包含

MVP 不做：

- 完整灾害应急系统。
- 火势蔓延仿真。
- 自动确认火灾。
- 自动灭火调度或消防指挥。
- 完整多部门联合指挥与审批流。
- 真实第三方反制设备自动实控。
- 通用规则引擎或 AI 自动决策平台。
- 全量实时流处理平台。

## 4. 统一事件模型

### 4.1 事件类型

MVP 事件队列至少支持：

| 事件类型 | 含义 |
|---|---|
| `airspace_intrusion` | 空域目标进入禁飞区或临时管控区 |
| `forest_fire_suspected` | 无人机巡查发现疑似林火，需要复核或通知 |

### 4.2 生命周期

所有正式空域事件共用生命周期：

```text
发现 → 待研判 → 已预警 → 处置编排中 → 处置执行中 → 已控制 → 已关闭 → 已复盘
```

说明：

- 候选事件记录不进入正式生命周期。
- 正式事件可以引用候选事件记录作为证据。
- 已关闭事件不能再认领，只能复盘。

### 4.3 通用字段语义

每个正式空域事件至少应表达：

- 事件类型
- 当前生命周期状态
- 风险等级
- 处置优先级
- 影响范围
- 相关空域网格
- 相关原始观测 / 融合判断
- 相关空域目标 / 目标航迹，若适用
- 相关巡查任务 / 火点线索，若适用
- 相关空域约束，若适用
- 相关研判因子
- 可用处置能力覆盖 / 应急能力
- 处置方案候选集
- 处置责任主体
- 协同单位，若有
- 关闭依据
- 决策证据链

## 5. 数据流

### 5.1 通用输入链路

```text
观测来源
  → 原始观测
  → 融合判断
  → 候选事件记录
  → 正式空域事件
  → 风险等级 / 处置优先级
  → 处置方案候选集
  → 事件历史 / 决策证据链
```

原则：

- 原始观测不可被覆盖删除。
- 融合判断可以更新，但必须保留证据、置信度和冲突信息。
- 外部厂商接口字段不能直接污染核心业务对象。
- 事件历史记录的不只是状态结果，还要记录决策证据链。

### 5.2 入侵事件数据流

```text
目标航迹 + 空域约束 + 空域网格
  → 网格粗筛
  → PostGIS 精确入侵判定
  → 入侵确认窗口
  → 候选事件记录 / airspace_intrusion
  → 风险分级规则
  → 处置能力覆盖匹配
  → 规则模板生成处置方案候选集
```

### 5.3 林火预警数据流

```text
巡查任务 + 无人机资产 + 巡查责任区
  → 热点/烟点/人工标注等火点线索
  → 原始观测
  → 融合判断
  → 火点确认窗口
  → 候选林火预警 / forest_fire_suspected
  → 林火风险分级规则
  → 林火复核与通知方案候选集
```

## 6. 空间与高度语义

### 6.1 空域网格

空域网格用于：

- 快速检索
- 多尺度聚合
- 邻域查询
- 风险叠加
- 态势表达
- 事件影响范围展示

但空域网格不作为唯一业务判定依据。精确判定仍使用 PostGIS 几何、时间范围和高度范围。

### 6.2 高度基准

空域约束、目标高度和事件证据必须显式记录高度基准：

- `AMSL`：海拔高程
- `AGL`：相对地面高度

要求：

- 不允许只记录“米”而不说明基准。
- Cesium 详情面板必须展示高度基准。
- GGER 3D 网格映射必须使用清晰一致的高度语义。
- 如果用户录入 AGL，计算前必须转换或解释为可计算的统一高度范围，并保留原始基准。

## 7. 风险等级与处置优先级

### 7.1 区分

- **风险等级**：事件对安全、任务、灾害现场或管控规则的威胁程度。
- **处置优先级**：平台应优先投入注意力、人员、设备或应急能力处理的程度。

无可用处置能力覆盖通常提高处置优先级，不直接提高风险等级，除非会导致风险后果扩大。

### 7.2 入侵风险分级

MVP 使用固定、可解释、可审计的风险分级规则，不做通用规则引擎。

建议规则：

- 禁飞区入侵：默认高风险。
- 临时管控区入侵：默认中高风险。
- 涉及核心保护区或高优先级约束：上调一级。
- 目标身份未知：上调一级。
- 观测证据冲突：进入待研判或降低自动确认等级。
- 无可用处置能力覆盖：提高处置优先级。

### 7.3 林火风险分级

林火疑似预警使用独立的林火风险分级规则，但输出统一风险等级和处置优先级。

因素包括：

- 火点线索置信度：明火 > 烟点 + 热点 > 单热点。
- 距离重点保护对象：居民点、景区、林区核心区、输电线路、重要设施。
- 环境研判因子：风速、风向、湿度、温度、可见度。
- 应急能力可达性：附近人员、车辆、无人机复核能力、通信覆盖。
- 巡查任务上下文：是否在计划巡查区内、是否重复出现。

## 8. 确认窗口

### 8.1 入侵确认窗口

目标航迹命中空域约束后，先形成候选事件；满足持续时间、连续观测点数或置信度阈值后，升级为正式入侵事件。

建议最小规则：

- 禁飞区：单点命中即可候选，连续 2 个观测点或持续 ≥ 3 秒升级。
- 临时管控区：连续 2 个观测点或持续 ≥ 5 秒升级。
- 目标速度/航向显示正在远离边界时，可保持待研判。
- 被过滤的候选事件保留轻量记录，便于误报分析。

### 8.2 火点确认窗口

火点线索先形成候选林火预警；满足多证据或持续性条件后升级为正式林火疑似预警。

建议最小规则：

- 单一热红外热点：候选林火预警。
- 热点 + 烟点同区域：升级为林火疑似预警。
- 热点持续 ≥ N 分钟或连续多帧出现：升级。
- 人工高置信标注：可直接升级。
- 与已知非火热源重叠：降低置信度或保持待复核。

## 9. 处置编排

### 9.1 通用边界

MVP 生成处置方案候选集，不输出唯一自动命令。

每个候选方案应说明：

- 适用事件
- 推荐动作
- 所需处置能力覆盖或应急能力
- 预计效果
- 风险与副作用
- 前置条件
- 置信度
- 排序原因

### 9.2 设备联动边界

MVP 不自动触发真实第三方反制动作。

允许：

- 读取设备状态、位置、覆盖、告警。
- 生成处置建议。
- 记录拟触发动作。
- 在演示环境中执行模拟联动。

不允许：

- 自动实控第三方反制设备。
- 绕过人工确认或授权来源。
- 缺少审计和中止机制的真实联动。

### 9.3 林火处置边界

林火 MVP 只覆盖复核、通知和升级。

允许：

- 派无人机资产二次复核。
- 调整巡查任务覆盖火点周边。
- 通知护林员或值班员核查。
- 推荐附近应急能力作为参考。
- 升级为人工应急处置流程。

不包含：

- 自动派遣消防队。
- 灭火物资调度。
- 火势扑救指挥。

## 10. 工作台设计原则

空域事件处置工作台不是普通综合态势驾驶舱，而是按事件处置组织。

首页优先展示：

1. 按处置优先级排序的事件队列。
2. 风险等级与排序原因。
3. 事件影响范围和相关空域网格。
4. 触发证据：目标、约束、火点线索、研判因子等。
5. 可用处置能力覆盖或应急能力。
6. 处置方案候选集。
7. 事件认领状态、责任主体和协同单位。
8. 执行状态、关闭依据和事件历史。

未认领事件应突出显示或提高处置优先级。

## 11. MVP 验收用例

### 11.1 禁飞/管控区入侵

验收输入：

- 一个已配置禁飞区或临时管控区。
- 一条模拟或历史目标航迹。
- 至少一个处置能力覆盖，或明确无可用能力。

验收步骤：

1. Cesium 显示空域约束范围和 GGER 空域网格。
2. 播放或导入目标航迹。
3. 目标进入约束范围时产生候选事件记录。
4. 满足入侵确认窗口后生成正式空域事件。
5. 生命周期至少流转：发现 → 待研判 → 已预警。
6. 风险等级展示触发原因、证据和影响范围。
7. 展示命中的空域网格和高度基准。
8. 匹配处置能力覆盖，或说明无可用能力。
9. 生成处置方案候选集。
10. 人工认领并选择处置结果：已控制 / 误报 / 持续观察。
11. 关闭事件并记录关闭依据与决策证据链。

### 11.2 无人机林火巡查预警

验收输入：

- 一个巡查任务和巡查责任区。
- 一架无人机资产及计划/实际航迹。
- 一组火点线索：热点、烟点、人工标注或模拟观测。
- 可选环境研判因子和附近应急能力。

验收步骤：

1. Cesium 显示巡查责任区、无人机航迹和空域网格。
2. 接收或模拟火点线索，并展示为原始观测。
3. 形成融合判断，展示置信度和证据来源。
4. 满足火点确认窗口后生成林火疑似预警事件。
5. 风险等级展示火点线索、影响对象、环境因子和应急能力可达性。
6. 处置优先级进入统一 MVP 事件队列排序。
7. 生成复核与通知类处置方案候选集。
8. 人工认领并选择：二次复核 / 通知核查 / 排除 / 升级处置。
9. 关闭或升级事件，并记录关闭依据与决策证据链。

## 12. 与现有仓库能力的关系

当前仓库已有基础：

- `airspace.no_fly_zone`
- `airspace.temp_control_zone`
- `height_datum` 语义与约束
- PostgREST CRUD 暴露脚本
- `citydb_grid.flight_obstacles`
- `list_flight_obstacles_gger(...)` RPC
- Cesium + GGER 空域网格可视化方案
- 飞行障碍物与路径规划相关文档/脚本

MVP 优先复用这些能力，先完成禁飞/管控区入侵闭环，再复用统一事件底座扩展无人机林火巡查预警闭环。

## 13. 建议实施顺序

1. 建立统一空域事件、事件历史、候选事件记录和证据链模型。
2. 接入现有禁飞区/临时管控区为空域约束。
3. 接入或模拟目标航迹。
4. 实现入侵判定：网格粗筛 + PostGIS 精确判定 + 入侵确认窗口。
5. 实现风险等级、处置优先级和规则模板候选方案。
6. 实现空域事件处置工作台的事件队列、详情、认领、关闭。
7. 建立巡查任务、巡查责任区和火点线索模型。
8. 实现火点确认窗口和林火风险分级规则。
9. 将林火疑似预警接入同一 MVP 事件队列和工作台。
