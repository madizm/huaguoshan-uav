# 适飞空域表结构说明

## 1. 概述

适飞空域数据保存于 PostgreSQL/PostGIS 的 `airspace.suitable_fly_zone` 表。每行对应源 KML 的一个 `Placemark`，用于表达无人机**允许飞行的水平范围**。

> 适飞空域不是禁飞区，也不进入 `citydb_grid.flight_obstacles` 障碍物链路。禁飞区和临时管制区分别使用 `airspace.no_fly_zone`、`airspace.temp_control_zone`。

当前数据由 `data/shifeikongyu.kml` 导入，使用脚本 `scripts/import_kml_suitable_fly_zones.py`。

## 2. 表定义

```sql
create table airspace.suitable_fly_zone (
  id bigint generated by default as identity primary key,
  name text not null,
  source_file text not null,
  source_feature_id text not null,
  source_layer text,
  source_properties jsonb not null default '{}'::jsonb,
  geom geometry(MultiPolygon, 4326) not null,
  imported_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  unique (source_file, source_feature_id)
);

create index suitable_fly_zone_geom_gix
  on airspace.suitable_fly_zone using gist (geom);
```

表注释：`适飞空域范围。每条记录对应源 KML 的一个 Placemark。`

## 3. 字段说明

| 字段 | 类型 | 可空 | 默认值 | 说明 |
|---|---|:---:|---|---|
| `id` | `bigint` | 否 | identity | 主键。|
| `name` | `text` | 否 | — | 适飞空域对应的江苏省地级市名称。优先取 KML `ExtendedData.aliasname`，没有时取 `Placemark.name`；取值见下表。|
| `source_file` | `text` | 否 | — | 导入源标识；默认是 KML 文件名，例如 `shifeikongyu.kml`。|
| `source_feature_id` | `text` | 否 | — | KML `Placemark/@id`，例如 `jssmerge.3`。与 `source_file` 共同唯一。|
| `source_layer` | `text` | 是 | `null` | KML `ExtendedData.layer`，用于保留源图层名称。|
| `source_properties` | `jsonb` | 否 | `'{}'::jsonb` | KML 的全部 `ExtendedData/SimpleData` 属性，作为导入溯源信息保存。|
| `geom` | `geometry(MultiPolygon, 4326)` | 否 | — | 适飞空域水平范围，WGS84 经纬度坐标；保留一个 Placemark 中的多个 Polygon 和内环（孔洞）。|
| `imported_at` | `timestamptz` | 否 | `now()` | 本次源数据写入时间。重复导入时更新。|
| `updated_at` | `timestamptz` | 否 | `now()` | 记录最后更新的时间。|

## 4. 约束与索引
## 4. 约束与索引

### 4.1 约束

| 名称 | 类型 | 定义 | 作用 |
|---|---|---|---|
| `suitable_fly_zone_pkey` | 主键 | `primary key (id)` | 唯一标识一条适飞空域记录。|
| `suitable_fly_zone_source_file_source_feature_id_key` | 唯一约束 | `unique (source_file, source_feature_id)` | 保障同一源文件的同一 KML 要素可幂等导入，避免重复记录。|

### 4.2 索引

| 名称 | 类型 | 字段 | 用途 |
|---|---|---|---|
| `suitable_fly_zone_pkey` | B-tree | `id` | 主键查询。|
| `suitable_fly_zone_source_file_source_feature_id_key` | B-tree | `source_file, source_feature_id` | 导入 upsert 冲突检查和源要素定位。|
| `suitable_fly_zone_geom_gix` | GiST | `geom` | `ST_Intersects`、`ST_Contains` 等空间范围查询。|

## 5. 数据语义

1. **坐标系**：KML 经纬度按 WGS84 写入，SRID 固定为 `4326`。
2. **空间维度**：源 KML 提供的是二维边界；即使坐标元组携带 KML altitude，导入时也不作为三维几何或高度约束保存。
3. **高度限制**：本表不含 `min_height`、`max_height` 或 `height_datum`。如业务需要按高度区分适飞空域，应另行设计垂直空域规则，避免把未提供的高度解释为无限高度范围。
4. **多面与孔洞**：同一 `Placemark` 的多个 `Polygon` 组成一个 `MultiPolygon`；`innerBoundaryIs` 会作为 Polygon 内环保留，不可飞的洞不会被填平。
5. **几何修复**：写入前使用 `ST_MakeValid`，并归一为 `MultiPolygon`，以适配 PostGIS 空间查询。

## 6. 导入与验证

### 6.1 导入命令

```bash
# 仅校验 KML，不写入数据库
uv run scripts/import_kml_suitable_fly_zones.py --dry-run

# 导入默认源 data/shifeikongyu.kml
uv run scripts/import_kml_suitable_fly_zones.py

# 指定数据源和稳定的源文件标识
uv run scripts/import_kml_suitable_fly_zones.py \
  --kml data/shifeikongyu.kml \
  --source-file shifeikongyu.kml
```

脚本会在目标不存在时创建 `airspace.suitable_fly_zone` 及上述索引；以 `(source_file, source_feature_id)` 执行 upsert，因此重复执行不会新增重复记录。

### 6.2 当前导入结果

`data/shifeikongyu.kml` 当前包含 13 个 `Placemark`，导入后为：

- 13 条适飞空域记录；
- 1267 个 Polygon；
- 1545 个内环；
- 所有写入的几何均通过 `ST_IsValid` 校验。

### 6.3 常用查询

```sql
-- 查询全部适飞空域及其多面数量
select
  id,
  name,
  source_layer,
  st_numgeometries(geom) as polygon_count
from airspace.suitable_fly_zone
order by id;

-- 查询包含某个 WGS84 坐标点的适飞空域
select id, name
from airspace.suitable_fly_zone
where st_covers(
  geom,
  st_setsrid(st_makepoint(119.0, 34.6), 4326)
);

-- 检查导入源的几何有效性
select id, name
from airspace.suitable_fly_zone
where source_file = 'shifeikongyu.kml'
  and not st_isvalid(geom);
```
