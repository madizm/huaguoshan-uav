# 低空空域事件处置平台

本项目围绕低空空域中的无人机目标、空域规则、环境要素、障碍物、反制设备与应急资源，支撑风险发现、研判预警、处置建议、资源调度和复盘。

## Language

**低空空域事件处置闭环**:
平台的主业务闭环：围绕低空空域中的风险目标或灾害场景，完成发现、研判、预警、处置建议、资源调度和复盘。
_Avoid_: 无人机管理平台、防控预警平台、灾害应急平台、智慧决策平台

**空域事件**:
需要平台研判或处置的业务对象。它可由目标、规则、环境、灾害、设备或人工上报触发，但只有需要预警、决策、调度、处置或复盘时才成立。
_Avoid_: 原始事件、数据变化、告警

**无人机资产**:
已登记、可被平台调度或管理的己方或授权无人机。
_Avoid_: 无人机、目标

**空域目标**:
被雷达、无线电、光电、人工等来源发现并进入跟踪或研判流程的空中对象，可能是未知无人机、授权无人机、鸟群、航模或误报。
_Avoid_: 无人机、资产

**预警**:
空域事件经过研判后形成的风险状态或提示结果，不是空域目标的固有属性。
_Avoid_: 目标风险、告警

**空域网格**:
用于归一化表达低空空域对象空间影响范围的北斗/GGER 网格单元或网格集合。它是平台的空间索引与态势聚合单元，原始对象仍保留真实坐标或几何。
_Avoid_: 地图格子、展示网格

**空域约束**:
某个时空范围内对飞行、通行、侦察或应急调度的限制条件。禁飞区、临时管控区、灾害应急区、环境风险区和普通业务建议区都属于空域约束。
_Avoid_: 禁飞区、规则、管控区

**飞行障碍**:
会阻挡、限制或改变目标航迹可行性的空间对象或空间范围。建筑、地形、空域约束和未来接入源都可以形成飞行障碍，但飞行障碍应保留来源可追溯性。
_Avoid_: 障碍图层、建筑物、禁飞区

**处置能力覆盖**:
某种探测、识别、跟踪、干扰、诱骗、迫降或捕获能力在特定时空范围内的有效覆盖。反制设备通常产生处置能力覆盖；只有影响己方飞行、通信或任务安全时才额外形成空域约束。
_Avoid_: 反制范围、设备覆盖区

**处置编排**:
平台围绕空域事件选择处置策略、匹配处置能力覆盖、生成操作建议或触发已授权联动动作的过程。它不默认包含对所有第三方反制设备的直接控制。
_Avoid_: 反制设备控制、设备遥控

**设备接入级别**:
平台对某个第三方设备在业务上被授权使用的最高协作程度，而不是单纯技术连接能力。包括可观测、可建议、可联动和可控制四级。
_Avoid_: API 能力、设备权限

**研判因子**:
会改变空域事件风险判断、处置可行性或应急调度结果的数据因素。环境要素、目标观测可信度、设备状态和资源可用性都可以成为研判因子。
_Avoid_: 背景图层、展示数据

**风险等级**:
空域事件经过研判后形成的可解释风险结果。它必须能追溯到触发原因、证据、影响范围、相关目标、相关约束、研判因子和建议动作。
_Avoid_: 展示颜色、告警颜色

**应急资源**:
可被调度参与处置的人员、物资、车辆、无人机资产、传感器或临时通信设备等对象。应急资源回答“有什么”。
_Avoid_: 库存、台账

**应急能力**:
由一个或多个应急资源在特定时空条件下形成的可执行处置能力。应急能力回答“能做什么、在哪里、何时可用、代价是什么”。
_Avoid_: 资源、物资

**观测来源**:
产生目标观测、环境观测、设备状态或告警的数据来源，如雷达、光电、无线电侦测、气象站、电磁监测、人工上报或第三方平台。观测来源回答“我知道了什么”。
_Avoid_: 传感器、设备

**态势快照**:
某一时刻平台对空域目标、空域约束、研判因子、处置能力覆盖和应急资源状态的综合表达，用于当前研判和大屏展示。
_Avoid_: 实时流、当前数据

**事件历史**:
空域事件从发现、研判、预警、处置建议、资源调度到复盘的状态变化和关键证据记录。
_Avoid_: 日志、流水

**空域事件处置工作台**:
围绕当前待处理空域事件组织的大屏与值班操作界面，以处置优先级排序，优先呈现事件、风险、证据、处置能力、应急能力、推荐动作和执行状态。
_Avoid_: 综合态势驾驶舱、数据大屏

**空域事件生命周期**:
空域事件从发现到复盘的业务状态流转：发现、待研判、已预警、处置编排中、处置执行中、已控制、已关闭、已复盘。
_Avoid_: 告警状态、处理状态

**关闭依据**:
空域事件被关闭时必须记录的可解释证据和原因。目标确认离开影响范围、约束不再被违反、风险持续降低、处置有效、人工确认误报或应急任务结束都可以成为关闭依据。
_Avoid_: 目标消失、自动关闭

**处置方案候选集**:
智慧决策输出的可解释处置方案排序，而不是自动唯一命令。每个候选方案应说明推荐动作、所需能力、预计效果、风险副作用、前置条件、置信度和排序原因。
_Avoid_: 唯一方案、自动决策

**处置责任主体**:
对空域事件当前研判、处置或复盘负责的人、队伍、组织或外部协同单位。系统推荐和自动联动都必须能追溯到授权来源与责任主体。
_Avoid_: 审批人、操作员

**灾害应急场景**:
空域事件处置闭环中的高优先级业务场景，而不是平行的独立事件体系。灾害可以形成空域事件，也可以改变其他空域事件的风险等级、空域约束、研判因子和应急能力可用性。
_Avoid_: 灾害应急模块、独立应急系统

**原始观测**:
外部来源在特定时间和空间范围内提供的未归一化数据记录，保留来源、原始载荷、可信度、接入通道和处理状态。原始观测回答“某来源说了什么”，不能直接替代平台业务对象。
_Avoid_: 业务数据、设备数据、接口数据

**融合判断**:
平台基于多个原始观测形成的当前可信解释，必须保留证据、置信度和冲突信息。融合判断可更新，但原始观测不可被覆盖删除。
_Avoid_: 覆盖更新、最新值

**禁飞/管控区入侵事件闭环**:
第一阶段最小可用闭环：当空域目标进入空域约束范围时，平台生成空域事件，计算风险等级，展示证据和影响范围，匹配可用处置能力覆盖，给出处置方案候选集，并记录事件历史。
_Avoid_: 全量平台、综合演示

**混合输入**:
MVP 阶段允许同时使用真实配置、人工录入、模拟轨迹、历史轨迹、静态数据和低频数据来验证事件闭环与领域模型。混合输入不代表事件历史可以伪造。
_Avoid_: 全模拟、全量真实接入

**高度基准**:
解释空域约束、目标高度和事件证据中高度数值含义的垂直参考语义，如 AMSL 表示海拔高程，AGL 表示相对地面高度。高度基准必须显式记录，不能只记录“米”。
_Avoid_: 高度、海拔、相对高度

**入侵判定**:
判断空域目标是否进入空域约束的业务判断。MVP 采用空域网格粗筛与 PostGIS 精确判定两阶段流程，空域网格不作为唯一判定依据。
_Avoid_: 网格命中、几何相交

**目标航迹**:
平台对同一个空域目标在一段时间内的连续跟踪结果，包含当前状态和历史观测。目标航迹是空域目标的运行时表达，不是裸点序列。
_Avoid_: 轨迹点、点序列

**入侵确认窗口**:
目标航迹命中空域约束后，从候选事件升级为正式空域事件所需的持续时间、连续观测点数或置信度条件。它用于降低边界抖动、定位误差和采样误差造成的误报。
_Avoid_: 立即告警、单点触发

**候选事件记录**:
目标航迹命中空域约束但尚未满足入侵确认窗口时产生的轻量记录，用于算法调试、误报分析和复盘追溯。候选事件记录不进入正式空域事件生命周期，但正式空域事件可以引用它作为证据。
_Avoid_: 空域事件、告警

**规则模板**:
MVP 阶段生成处置方案候选集的可解释规则集合，按事件类型、风险等级、约束类型、处置能力覆盖和设备接入级别组合推荐方案。不等同于复杂 AI 或全局优化。
_Avoid_: AI 决策、自动优化

**模拟联动**:
在不触发真实第三方反制动作的前提下，验证处置编排、授权记录、动作状态和事件历史的演示或测试联动。MVP 可使用模拟联动表达拟触发动作，但不代表真实实控能力。
_Avoid_: 自动反制、真实控制

**风险分级规则**:
MVP 阶段用于生成风险等级的固定、可解释、可审计规则集合。它可以存放在配置表或配置文件中，但不是面向用户开放的通用规则引擎。
_Avoid_: 规则引擎、展示颜色配置

**处置优先级**:
平台应优先投入注意力、人员、设备或应急能力处理某个空域事件的程度。它不同于风险等级；无可用处置能力覆盖通常提高处置优先级，而不直接提高风险等级。
_Avoid_: 风险等级、告警等级

**事件认领**:
处置责任主体对某个空域事件承担当前主处理责任的工作流动作。认领、转派和释放都必须写入事件历史；未认领事件应在工作台中突出显示或提高处置优先级。
_Avoid_: 查看事件、处理人

**协同单位**:
参与某个空域事件处置但不承担主处理责任的外部单位、队伍或组织。MVP 中协同单位只做轻量记录，不包含完整多单位审批或联合指挥流程。
_Avoid_: 主责任主体、协同流程

**决策证据链**:
事件历史中支撑关键状态变化、风险判断、处置编排、人工选择和关闭依据的证据集合。它应关联原始观测、融合判断、空域约束、入侵判定、风险等级、处置优先级、处置方案候选集和责任主体变化。
_Avoid_: 状态日志、操作记录

**无人机林火巡查预警闭环**:
MVP 中的应急预警场景：无人机资产执行林区巡查任务，基于热红外热点、可见光烟点、人工标注或历史轨迹形成融合判断，满足确认条件后生成林火疑似预警空域事件，并完成风险研判、处置建议、人工确认、关闭或升级。
_Avoid_: 灾害应急系统、林火业务平台

**林火疑似预警**:
无人机林火巡查预警闭环中的空域事件类型，表示平台基于热点、烟点、人工标注或融合判断认为某处可能发生林火，需要复核、通知或处置。
_Avoid_: 火情、火灾确认、无人机告警

**疑似火点复核闭环**:
无人机林火巡查预警闭环的业务边界：平台发现疑似火点并触发复核、通知或升级处置，但不自动确认火灾。火情确认需要人工、二次巡查、地面人员或权威系统确认。
_Avoid_: 自动火灾确认、火灾识别系统

**火点线索**:
无人机巡查、传感器或人工上报发现的可能与林火相关的观测对象，例如热红外热点、可见光烟点、明火疑似区域、异常温升或人工标注点。火点线索不是火灾确认。
_Avoid_: 火灾、火情

**巡查任务**:
无人机资产在指定时间窗口内，对某个林区、空域网格、区域或航线执行观测的轻量业务任务。巡查任务关联计划/实际航迹、任务状态、无人机资产，以及产出的原始观测或火点线索。
_Avoid_: 飞行任务、航线播放

**巡查责任区**:
巡查任务面向的林区、保护区、人工绘制区域或空域网格范围。巡查责任区保留业务边界表达，但必须映射到空域网格，以便与目标航迹、空域约束、火点线索和应急能力统一叠加。
_Avoid_: 林区图层、独立空间体系

**火点确认窗口**:
火点线索从候选林火预警升级为正式林火疑似预警所需的多证据、持续时间、连续帧数或人工高置信标注条件。它用于降低反光、热源设备、云影、图像噪声和农事用火造成的误报。
_Avoid_: 单点火警、立即确认

**林火风险分级规则**:
用于林火疑似预警的固定、可解释、可审计风险分级规则。它基于火点线索置信度、影响对象、环境研判因子、应急能力可达性和巡查任务上下文，输出统一的风险等级与处置优先级。
_Avoid_: 入侵风险规则、火灾等级

**林火复核与通知**:
MVP 中林火疑似预警的处置边界：派无人机资产二次复核、调整巡查任务、通知护林员或值班员核查、推荐附近应急能力作为参考，必要时升级为人工应急处置流程；不包含自动灭火调度或扑救指挥。
_Avoid_: 灭火调度、消防指挥

**MVP 事件队列**:
MVP 中禁飞/管控区入侵事件与林火疑似预警共用的空域事件处置队列。两类事件通过事件类型区分，但共用风险等级、处置优先级、事件认领、处置责任主体、处置方案候选集、关闭依据和决策证据链。
_Avoid_: 入侵模块、林火模块、独立事件列表
