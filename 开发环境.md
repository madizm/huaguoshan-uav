## postgrest

server: <http://10.1.109.151:13000/>

配置文件 `pgrest.conf` 只保留非密配置；数据库连接和 JWT 密钥通过环境变量或进程管理器注入，示例见 `pgrest.env.example`。

安装/收敛权限模块：

```bash
psql "$DATABASE_URL" -f backend/create_postgrest_authorization.sql
```

PostgREST 不应使用数据库超级用户连接；使用授权脚本创建的专用登录角色：

```bash
export PGRST_DB_URI="postgres://postgrest_authenticator:${PGRST_DB_PASSWORD}@10.1.109.151:5432/huaguoshan_projd"
export PGRST_JWT_SECRET="${HGS_POSTGREST_JWT_SECRET}"
postgrest pgrest.conf
```

初始化首个管理员账号时从环境变量读取密码并生成 Argon2id hash，不提交默认密码：

```bash
HGS_ADMIN_PASSWORD='从密码库获取' \
  uv run scripts/seed_platform_identity.py \
  --username admin \
  --display-name '值班管理员' \
  | psql "$DATABASE_URL"
```

重复执行权限安装命令和账号初始化命令应成功完成；账号初始化使用 `on conflict (username) do update` 收敛同一管理员身份，可用于验证脚本对已有数据库的幂等性。
